Step 1 · Draft
Draft stage - first translation

This stage creates the first translation from the source rows. Attach the glossary here when you want terms to be translated consistently from the start.

Draft instructions

Tell the AI how to create the first translation. Keep this focused on accuracy, structure, and source preservation.

Use this prompt to request the first translation. Important variables: {{lang}} {{hotelName}} {{glossaryRules}} {{rows}} {{translateHeader}}

Usually hidden. Use only when you want to change the AI role, translation behavior, or global draft rules. Vars: {{lang}}{{hotelName}}

Glossary - exact translations before Draft

Define exact translations for terms that must stay consistent. Example: whenever "Aparthotel" appears, translate it the same way.

Use this for terms that must have one consistent translation. This is connected to the Draft step through {{glossaryRules}}.
Simple mode: write the source term on the left and the required translation on the right. No JSON needed.

Expected structure: {"Aparthotel":"Aparthotel", "Serviced apartments":"Serviced Apartments"}

Step 2 · Polish
Polish stage - improve the draft

This stage receives the draft translation and improves wording, tone, and naturalness without changing facts. Attach language notes and terminology here.

Polish instructions

Tell the AI how to polish the draft. Focus on natural language, brand tone, and factual consistency.

Use this prompt to improve the draft after translation. Important variables: {{lang}} {{hotelName}} {{languageNotes}} {{strictTerminology}} {{sourceRows}} {{draftJson}}

Usually hidden. Use only when you want to change the AI role or global polish behavior. Vars: {{lang}}{{hotelName}}

Language Notes - style for Polish

Define how the language should sound: formal, natural, concise, hospitality-focused, local, or any special wording rules.

These notes are injected into the Polish step per language through {{languageNotes}}.
Terminology - wording to avoid after Draft

Define wording to avoid and what to use instead. Example: do not use "Haus"; use "Hotel".

Use this when the AI writes wording you do not want. This is connected to the Polish step through {{strictTerminology}}.
Simple mode: write what not to use, what to use instead, and why.

Expected structure: {"mappings":[{"forbidden":"Haus","preferred":"Hotel","reason":"Avoid house wording"}], "examples":[]}

How to Guide the AI

▼
This guide is intentionally at the bottom. Configure the tool first, then use this section when you need to understand what each area controls.

Draft stage

This creates the first translation.

  • Write instructions about translating every cell.
  • Ask it to preserve rows, columns, hotel names, numbers, URLs and codes.
  • Use {{glossaryRules}} so fixed terms affect the first draft.

Glossary inside Draft

Use this for exact translations.

  • Write how terms should be translated consistently.
  • Example: Aparthotel = Aparthotel.
  • Best for fixed names, amenities, booking terms and product names.

Polish stage

This improves the draft without changing facts.

  • Ask for natural, professional hotel wording.
  • Keep facts identical to the source.
  • Use {{sourceRows}} and {{draftJson}}.

Terminology inside Polish

Use this when the AI writes wording you do not want.

  • Write what to avoid.
  • Write what to use instead.
  • Add a short reason so the model understands the rule.

Language Notes inside Polish

Use this for style and tone.

  • Explain how the language should sound.
  • Example: formal German, natural Hebrew, concise Italian.
  • Do not use this for exact replacements. Use Glossary or Terminology instead.

Advanced prompts

System prompts are hidden by default because most users should not edit them.

  • Open them only when you want to change the AI role.
  • Keep the user prompts as the main area for normal instructions.
  • Advanced JSON is available for users who understand the internal structure.
